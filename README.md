# GitHub Pages

Структура проекта уже подготовлена для публикации как статический сайт:

- `index.html` — главная страница сайта
- `assets/` — все картинки, стили и скрипты
- `.nojekyll` — отключает Jekyll на GitHub Pages, чтобы ассеты с `_` в имени не ломались

Как опубликовать:

1. Создайте новый репозиторий на GitHub.
2. Загрузите в него содержимое этой папки.
3. На GitHub откройте `Settings` -> `Pages`.
4. В разделе `Build and deployment` выберите:
   `Source` -> `Deploy from a branch`
5. Выберите ветку:
   `main`
6. Выберите папку:
   `/ (root)`
7. Нажмите `Save`.

Через пару минут сайт будет доступен по адресу вида:

`https://USERNAME.github.io/REPOSITORY/`

Важно:

- Если репозиторий называется не `USERNAME.github.io`, сайт будет открываться с названием репозитория в URL.
- Музыка и часть внешних ресурсов у вас загружаются по внешним ссылкам. Если эти ссылки когда-нибудь перестанут работать, соответствующие элементы на сайте тоже перестанут загружаться.
